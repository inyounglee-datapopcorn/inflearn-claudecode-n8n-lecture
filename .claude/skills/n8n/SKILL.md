# n8n 워크플로우 관리 스킬

이 스킬은 n8n 워크플로우의 생성, 수정, 운영, 교육을 위한 통합 가이드입니다.
사용자의 요청 의도를 파악한 뒤, 아래 라우팅 테이블에 따라 해당 모듈 파일을 로드하여 작업을 수행하세요.

## 1. 모듈 라우팅 테이블

| 사용자 의도 | 라우팅 대상 |
|---|---|
| 워크플로우 생성 / 수정 / 분석 / 개선 | `references/workflow-builder.md` |
| 실행 확인 / 실패 원인 / 상태 점검 / 운영 리포트 | `references/ops-manager.md` |
| 튜토리얼 / 강의자료 / 실습 가이드 생성 | `references/teaching-guide.md` |
| 복합 요청 | 모든 모듈을 로드 (우선순위: ops → builder → teaching) |

## 2. 공통 규칙

모든 모듈이 공통으로 지켜야 할 원칙입니다.

### 시작 전 항상 실행
- **`n8n_health_check`** 를 먼저 호출하여 서버 상태를 확인합니다. 서버가 응답하지 않으면 작업을 중단하고 사용자에게 알립니다.

### 노드 검색 패턴
노드를 사용해야 할 때는 항상 다음 순서를 따릅니다.
1. `search_nodes` — 키워드로 후보 노드 검색
2. `get_node` — 선택한 노드의 상세 스펙 조회
3. `validate_node` — 노드 구성이 올바른지 검증

### 안전 원칙
- **파괴적 작업(삭제/수정) 전에는 반드시 사용자 확인을 받습니다.**
- **수정 전에는 `n8n_workflow_versions`로 현재 버전을 기록**하여 롤백 지점을 확보합니다.
- **API 키, 비밀번호, 토큰 등 비밀 정보를 절대 출력하지 않습니다.**
- 도구 사용법이 불확실할 때는 `tools_documentation`을 확인합니다.
- 모든 작업이 끝난 후 결과를 사용자에게 **요약 보고**합니다.

## 3. 주요 MCP 도구 목록

| 도구 이름 | 용도 |
|---|---|
| `n8n_create_workflow` | 새 워크플로우 생성 |
| `n8n_update_workflow` | 기존 워크플로우 수정 |
| `n8n_delete_workflow` | 워크플로우 삭제 |
| `n8n_list_workflows` | 워크플로우 목록 반환 |
| `validate_workflow` | 워크플로우 JSON 검증 |
| `n8n_execute_workflow` | 워크플로우 테스트/실행 |
| `search_nodes` | 사용할 노드 검색 |
| `get_node` | 노드 상세 스펙 조회 |
| `n8n_health_check` | 서버 상태 확인 |
